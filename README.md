# Academic Intervention SorterA modular desktop analytics platform designed to support higher-education intervention workflows, campaign management, faculty reporting, and operational analytics.Academic Intervention Sorter processes progress reports, midterm grade reports, intervention assignments, and campaign data into structured Excel workbooks with executive summaries, visual analytics, trend reporting, and semester-level tracking.Built with Python and Tkinter, the application was designed around real-world higher education advising operations and institutional reporting workflows.---# OverviewAcademic Intervention Sorter was developed to streamline intervention campaign workflows commonly performed by advising and student success teams.The application allows users to:* Process at-risk student data* Assign students into prioritized intervention groups* Track intervention campaigns across a semester lifecycle* Generate executive-ready Excel reporting workbooks* Analyze trends across multiple checkpoints* Monitor faculty progress report completion rates* Produce clean outreach and operational datasetsThe system emphasizes:* Operational efficiency* Workflow automation* Reporting quality* Data organization* Institutional analytics* Maintainable modular architecture---# Key Features## Intervention Campaign Processing* Processes at-risk student reports* Assigns students into intervention groups using configurable priority order* Prevents duplicate assignment across groups* Preserves all related course records for matched students* Supports reusable campaign configurations* Tracks intervention runs by semester and checkpoint## Midterm & Progress Report Analytics* Processes progress report and midterm grade exports* Identifies and organizes at-risk students* Supports multi-stage intervention workflows* Tracks campaign performance across checkpoints* Produces structured Excel reporting outputs## Trend & Recovery Analysis* Semester-level trend analysis* PR1 → Midterm → PR2 trajectory tracking* Recovery and persistence reporting* Group-level intervention analysis* Cross-checkpoint student movement analysis* Executive summary reporting with charts## Faculty Reporting & Completion Monitoring* Faculty progress report completion dashboards* College and department-level completion reporting* Faculty outreach download generation* Executive-style KPI summaries* Completion rate visualizations## Reporting & Export System* Multi-sheet Excel workbook generation* Dashboard-style summary tabs* Charts and visual reporting* QA logs and processing manifests* Workbook navigation/index tabs* Contact coverage analysis* Download-ready outreach datasets## Demo Mode* Included synthetic demo data* Fully functional sample workflows* Safe portfolio demonstration environment* Example outputs included---# ArchitectureThe application uses a modular processing architecture designed for maintainability and workflow separation.## Core Components### GUI LayerTkinter-based desktop interface with modular tab architecture:* Progress Report Processing* Midterm Processing* Trend Analysis* Campaign Management* Faculty Report Status* Settings & Configuration* Help & Demo Tools### Processing LayerDedicated processing modules handle:* Data ingestion* Validation* Mapping* Filtering* Assignment logic* Trend analysis* Campaign lifecycle management* Workbook generation### Export LayerSpecialized exporters generate:* Summary workbooks* Trend reports* Faculty completion reports* Semester reports* Campaign analytics### UtilitiesShared utility modules provide:* Excel formatting* Validation* Logging* Configuration management* Semester persistence* QA tracking---# Example Workflow## Intervention Campaign Workflow1. Load:   * Progress report or midterm report   * Contact report   * Intervention group files2. Configure:   * Group processing order   * Semester/checkpoint   * Campaign settings3. Run processing pipeline4. Generate:   * Intervention group workbooks   * Summary dashboards   * QA logs   * Trend reports   * Faculty reporting outputs5. Export executive-ready Excel reports---# Sample OutputsGenerated workbooks include:* Executive summary dashboards* Group assignment tabs* Trend analysis reports* Faculty completion
